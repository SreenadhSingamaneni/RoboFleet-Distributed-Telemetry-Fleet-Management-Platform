# Contributing

1. Create a focused branch from `main`.
2. Add or update tests with every behavior change.
3. Run `make test` and `docker compose config` before opening a pull request.
4. Explain operational impact, schema changes, and rollback steps in the pull request.
5. Never commit `.env`, cloud credentials, patient information, or robot secrets.

Commits use imperative subjects, for example: `Add stalled-robot alert rule`.
